🔍 Scanning de vulnerabilidades: concepto y propósito

El scanning de vulnerabilidades es un proceso automatizado que permite identificar debilidades de seguridad en sistemas, redes, aplicaciones y servicios. A diferencia de una auditoría manual, las herramientas de scanning recorren cientos o miles de activos en cuestión de horas, detectando configuraciones inseguras, software desactualizado, puertos abiertos innecesarios, credenciales débiles y servicios mal configurados. Este proceso es fundamental en cualquier programa de ciberseguridad porque proporciona una visión general del estado de salud de la infraestructura y ayuda a priorizar los parches críticos.

Los scanners más avanzados utilizan bases de datos actualizadas de vulnerabilidades conocidas, como el sistema CVE (Common Vulnerabilities and Exposures) y el estándar CVSS (Common Vulnerability Scoring System) que asigna una puntuación de gravedad (de 0 a 10) a cada fallo. Ejemplos de herramientas líderes son Nessus (muy usado en entornos corporativos), OpenVAS (alternativa open source), QualysGuard (solución cloud) y Nexpose de Rapid7, que se integra directamente con Metasploit para facilitar pruebas de penetración.

⚙️ Metasploit Framework: arquitectura y componentes

Metasploit Framework (MSF) es una plataforma modular escrita en Ruby que se ha convertido en el estándar de facto para pruebas de penetración y desarrollo de exploits. Su arquitectura basada en módulos permite una gran flexibilidad: exploits (código que aprovecha una vulnerabilidad específica), payloads (código que se ejecuta tras el exploit, como shells reversas o descarga de malware), auxiliares (escáneres, fuzzing, ataques de fuerza bruta) y post-explotación (técnicas para mantener acceso, extraer hashes, pivoting).

Entre los payloads más conocidos están meterpreter (shell avanzada con capacidades de inyección de comandos, captura de pantalla, descarga de archivos) y reverse_tcp (shell reversa que evade firewalls). Metasploit también incluye herramientas de evasión de antivirus y generación de payloads ofuscados. Los profesionales de seguridad lo utilizan para simular ataques realistas, evaluar la efectividad de los controles defensivos y desarrollar firmas para IDS/IPS.

msfconsole > use exploit/windows/smb/ms17_010_eternalblue
set RHOSTS 192.168.1.100
set PAYLOAD windows/x64/meterpreter/reverse_tcp
exploit

El ejemplo anterior muestra cómo se lanza el famoso exploit EternalBlue (MS17-010) contra un sistema Windows vulnerable, obteniendo una shell reversa. La versatilidad de Metasploit lo hace indispensable en pruebas de intrusión autorizadas.

⚠️ Explotación local vs remota: diferencias y técnicas

La explotación remota ocurre cuando el atacante no tiene acceso previo al sistema objetivo y se aprovecha de servicios expuestos en red (por ejemplo, un servidor web vulnerable a inyección SQL o RCE). En este escenario, el objetivo suele ser obtener una primera foothold dentro del entorno. Por otro lado, la explotación local asume que el atacante ya tiene algún nivel de acceso (por ejemplo, una cuenta de usuario limitada) y busca elevar privilegios a root o SYSTEM, escapar de contenedores o jailbreaks, o persistir en el sistema. Las técnicas de escalada local incluyen: explotación de binarios SUID en Linux, vulnerabilidades de kernel, abuso de tareas programadas mal configuradas, o uso de herramientas como Mimikatz para extraer credenciales de memoria.

Las señales de una explotación remota en curso incluyen picos anómalos de tráfico hacia endpoints poco comunes, errores con stack traces visibles en respuestas HTTP, nuevas cuentas de usuario inesperadas, conexiones salientes a IPs sospechosas, y alertas de IDS/IPS por patrones de inyección o shells reversas.

🕵️ Persistencia, backdoors y shell reversa

Persistencia se refiere al conjunto de técnicas que utiliza un atacante para asegurar que su acceso al sistema comprometido se mantenga incluso después de reinicios, cierres de sesión, aplicación de parches o intentos de limpieza por parte del administrador. Los métodos comunes incluyen: creación de servicios maliciosos, tareas programadas (cron o scheduled tasks), modificación de scripts de inicio, inyección en el registro de Windows, o uso de backdoors a nivel de firmware (UEFI/BIOS) que sobreviven a formateos completos.

Una shell reversa es un tipo de backdoor donde la máquina víctima inicia la conexión hacia el atacante (en lugar de esperar conexiones entrantes). Esto permite evadir firewalls y NAT, ya que el tráfico saliente suele estar menos restringido. La shell reversa puede programarse para reconectarse periódicamente (por ejemplo, cada 60 segundos) y camuflarse como tráfico legítimo utilizando puertos 80 (HTTP) o 443 (HTTPS), o incluso encapsulada en peticiones DNS. Herramientas como Netcat, Metasploit o PowerCat permiten establecer shells reversas de forma sencilla.

🧹 Limpieza de huellas y manipulación de logs

Después de lograr sus objetivos, un atacante experimentado intenta borrar, alterar o deshabilitar las evidencias de su actividad para dificultar el análisis forense y retrasar la respuesta del equipo de seguridad. Las técnicas incluyen: borrar entradas específicas de logs (por ejemplo, eventos de inicio de sesión, creación de procesos, ejecución de comandos), desactivar servicios de auditoría (como el servicio de eventos de Windows o syslog en Linux), modificar timestamps de archivos para que parezcan legítimos, y usar rootkits para ocultar procesos, archivos y conexiones de red.

Los logs son la principal fuente de evidencia forense. Contienen información crítica: timestamp, usuario, origen IP, tipo de evento, mensaje de error o éxito. Un sistema de eventos es más amplio que un simple archivo de log: incluye la plataforma que gestiona, almacena y permite consultar los registros (ej. Windows Event Viewer, ELK stack, syslog-ng). Un atacante que desactiva el servicio de eventos está llevando a cabo una clara acción de limpieza de huellas.

🌐 Ataque Man-in-the-Middle (MitM) con Ettercap

El ataque Man-in-the-Middle consiste en interponerse en la comunicación entre dos partes (por ejemplo, un usuario y su router, o un cliente y un servidor web) sin que estas lo detecten. El atacante puede interceptar, modificar o redirigir el tráfico. Las técnicas más comunes son el ARP spoofing (en redes locales, el atacante asocia su dirección MAC con la IP de la puerta de enlace) y el DNS spoofing (redirige consultas DNS a sitios maliciosos).

Ettercap es una herramienta open source que facilita estos ataques con una interfaz tanto gráfica como por comandos. Permite sniffing de contraseñas en texto claro, inyección de código en páginas web, captura de cookies y sesiones, y ataques SSL strip. Las consecuencias de un MitM exitoso incluyen robo de credenciales bancarias, suplantación de identidad, espionaje industrial y manipulación de información crítica. Para defenderse, se recomienda el uso de cifrado punto a punto (HTTPS, SSH, VPN), implementación de ARP estático o detección con herramientas como XArp, y segmentación de red mediante VLANs.

# Ejemplo básico de ARP spoofing con Ettercap
sudo ettercap -T -M arp:remote /192.168.1.1// /192.168.1.100//
# Esto envenena la tabla ARP entre el gateway (192.168.1.1) y la víctima (192.168.1.100)
Seguridad Perimetral

Firewalls · Zona Desmilitarizada · IDS/IPS · Cisco ASA

🔥 Firewall: definición, tipos y filosofías

Un firewall es un sistema de seguridad (hardware o software) que controla el tráfico de red entrante y saliente basándose en un conjunto de reglas predefinidas. Su función principal es actuar como una barrera protectora entre redes confiables (por ejemplo, la red interna de una organización) y redes no confiables (como Internet). Existen varios tipos según su nivel de funcionamiento: filtrado de paquetes (analiza cabeceras IP, puerto y protocolo, es rápido pero limitado), stateful inspection (mantiene una tabla de conexiones activas y toma decisiones contextuales), y firewalls de aplicación o proxy (filtran a nivel de aplicación, por ejemplo inspeccionando comandos HTTP o SMTP, ofrecen mayor seguridad pero consumen más recursos).

Además, según su ubicación podemos hablar de firewalls personales (como el firewall de Windows), perimetrales (dispositivos dedicados en el borde de la red) y firewalls en la nube. Las filosofías de configuración determinan cómo se comporta el firewall por defecto: "todo permitido, excepto lo prohibido" es menos restrictivo y más arriesgado; "todo prohibido, excepto lo permitido" es la más recomendada en entornos críticos porque solo se habilita lo necesario. Esta última sigue el principio de privilegio mínimo.

🌐 Zona Desmilitarizada (DMZ): concepto y arquitectura

Una Zona Desmilitarizada (DMZ) es una subred aislada y controlada que actúa como área intermedia entre la red interna privada (confiable) y redes externas no confiables (Internet). Su nombre proviene de la analogía militar: así como una zona desmilitarizada es un espacio neutral entre dos territorios en conflicto, en redes la DMZ funciona como un espacio de contención. En ella se ubican servicios que deben ser accesibles desde el exterior, como servidores web, de correo electrónico, DNS o FTP.

El principal objetivo de una DMZ es reducir la superficie de ataque y aumentar la seguridad. Si un atacante logra comprometer un servidor en la DMZ, el diseño de la red limita su capacidad de moverse hacia la red interna, ya que el firewall restringe estrictamente el tráfico de DMZ a LAN. Normalmente se emplean dos o tres firewalls (o un firewall con múltiples interfaces) para regular los flujos: Internet ↔ DMZ (permite acceso público a los servicios), DMZ ↔ Red interna (muy restringido, solo conexiones necesarias y bajo auditoría), y Red interna ↔ Internet (salida controlada).

⚠️ IPS vs IDS: prevención y detección de intrusiones

La seguridad perimetral no se completa solo con firewalls; es necesario también detectar y responder ante ataques que logran atravesar las defensas. Aquí entran dos tecnologías complementarias: IDS (Intrusion Detection System) y IPS (Intrusion Prevention System). El IDS monitorea el tráfico o la actividad del sistema, compara con firmas o detecta anomalías, y genera alertas o registra eventos, pero no bloquea automáticamente el tráfico malicioso. Es una herramienta pasiva, muy útil para auditorías y análisis forense.

Por otro lado, el IPS se ubica en línea (inline) dentro del flujo de datos y, además de detectar, actúa automáticamente para bloquear, descartar paquetes o cortar conexiones en tiempo real. Existen varios tipos: NIPS (Network IPS) analiza tráfico agregado de red, HIPS (Host IPS) protege procesos y archivos en un servidor específico, WIPS (Wireless IPS) protege redes Wi-Fi, y NBA (Network Behavior Analysis) se enfoca en identificar anomalías en el comportamiento del tráfico. El uso conjunto de firewall, IPS e IDS proporciona una defensa en profundidad robusta.

⚙️ Configuración básica en Cisco ASA

Los firewalls Cisco ASA (Adaptive Security Appliance) son ampliamente utilizados en entornos empresariales. A continuación se muestran comandos esenciales para configurar las interfaces, niveles de seguridad, NAT y reglas de acceso:

interface GigabitEthernet0/0
nameif inside
security-level 100
ip address 192.168.1.1 255.255.255.0
no shutdown

interface GigabitEthernet0/1
nameif outside
security-level 0
ip address 200.100.50.1 255.255.255.0
no shutdown

object network RED_INTERNA
subnet 192.168.1.0 255.255.255.0
nat (inside,outside) dynamic interface

access-list OUTSIDE_IN extended permit tcp any any eq 80
access-list OUTSIDE_IN extended permit tcp any any eq 443
access-group OUTSIDE_IN in interface outside

El security-level va de 0 (menos confiable) a 100 (más confiable). Por defecto, el tráfico de inside hacia outside está permitido, pero el iniciado desde outside hacia inside se bloquea a menos que se defina una ACL explícita. El NAT dinámico permite que los equipos internos salgan a Internet usando la IP pública de la interfaz outside. Las ACL (Access Control Lists) permiten tráfico específico, como navegación web (puertos 80 y 443) desde cualquier origen hacia el exterior.

En un laboratorio con Cisco Packet Tracer, estos comandos permiten simular una red corporativa segura con firewall perimetral, demostrando los principios fundamentales de filtrado de paquetes y políticas de seguridad.

🔐 Buenas prácticas y recomendaciones finales

Para una seguridad perimetral efectiva se recomienda: (1) Adoptar la filosofía "todo prohibido, excepto lo permitido" en firewalls, (2) implementar una DMZ para todos los servicios públicos, (3) combinar firewall con IPS para bloqueo automático y con IDS para monitoreo y auditoría, (4) mantener actualizadas las bases de firmas de los sistemas de detección, (5) aplicar el principio de mínimo privilegio en las reglas de acceso, (6) realizar pruebas de penetración periódicas y (7) capacitar al personal en la gestión de eventos y logs. La seguridad perimetral no es estática; requiere revisión constante y adaptación a nuevas amenazas.
