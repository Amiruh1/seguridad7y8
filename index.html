Semana 7

Scanning · Metasploit · Explotación

¿Qué es el Scanning de Vulnerabilidades?

Proceso automatizado que identifica debilidades en sistemas, redes, aplicaciones o servicios. Detecta fallos de seguridad conocidos —configuraciones incorrectas, servicios inseguros, software desactualizado— antes de que sean explotados.

Características del Scanning

  • Automatización: No requiere intervención manual en cada prueba.
  • Bases de datos: Usan CVE, CVSS y fuentes actualizadas.
  • Rapidez: Analizan gran cantidad de sistemas en poco tiempo.
  • Reportes: Resultados organizados y priorizados por nivel de riesgo.

Herramientas Comunes de Scanning

  • Nessus: Muy usada en entornos corporativos, interfaz amigable.
  • OpenVAS: Alternativa open-source de alto nivel (Greenbone).
  • Nexpose (Rapid7): Integración con Metasploit.
  • QualysGuard: Solución cloud para grandes infraestructuras.
  • Nikto: Scanner especializado en aplicaciones web.

Ventajas y Limitaciones del Scanning

✅ Ventajas

  • Ahorra tiempo frente a revisiones manuales
  • Cobertura amplia en redes grandes
  • Facilita la priorización de parches

⚠ Limitaciones

  • Puede generar falsos positivos
  • No sustituye al análisis manual o pentesting
  • Requiere actualizaciones constantes de su BD

Metasploit Framework (MSF)

Plataforma modular escrita en Ruby para desarrollar, probar y ejecutar exploits. Usada por profesionales de seguridad en pruebas de penetración, investigación de vulnerabilidades y desarrollo de exploits.

  • Exploits: Código que aprovecha una vulnerabilidad específica para obtener acceso o control.
  • Módulos: Componentes personalizables para distintos escenarios.
  • Payloads: Código ejecutado en el objetivo tras el exploit (ej. shell, malware).
  • Auxiliares: Herramientas de reconocimiento y escaneo de redes.
  • Post-explotación: Técnicas para mantener acceso y extraer datos.

Usos Principales de Metasploit

  • Pruebas de penetración: Simular ataques para identificar debilidades y fortalecer defensas.
  • Desarrollo de firmas IDS: Crear firmas que detecten y bloqueen intentos de ataque.
  • Investigación de vulnerabilidades: Explorar cómo funcionan para desarrollar soluciones.

Explotación Local vs. Remota

LOCAL

El atacante ya tiene acceso (cuenta de usuario, ejecución local) y aprovecha fallas para escalar privilegios, escapar de confinamientos o mantener acceso.


  • Escalada de privilegios (root/admin)
  • Persistencia
  • Evasión y ocultamiento
  • Acceso a datos protegidos

REMOTA

Sin acceso físico ni sesión local previa. El atacante interactúa con un servicio o protocolo expuesto logrando ejecución, filtrado de datos o control del sistema.


  • Ejecución Remota de Código (RCE)
  • Inyección SQL (SQLi)
  • Vulnerabilidades de día cero
  • Software sin parches

Señales de Explotación Remota en Curso

  • Picos de tráfico anómalo a un endpoint
  • Errores con stack traces visibles en respuestas HTTP
  • Comportamiento extraño en bases de datos (consultas inusuales)
  • Nuevas cuentas o archivos en servidores
  • Alertas de IDS/IPS por patrones de inyección o shells
  • Comunicación saliente a IPs desconocidas
Semana 8

Persistencia · Limpieza de Huellas · MitM

Persistencia

Conjunto de técnicas que un atacante usa para mantener acceso a un sistema comprometido, incluso después de reinicios, cierres de sesión, actualizaciones o intentos de limpieza del administrador.

  • Sobrevive a reinicios: El código se vuelve a ejecutar al iniciar el sistema.
  • Difícil de detectar: Imita procesos legítimos o usa configuraciones normales.
  • Múltiples niveles: Usuario, SO, aplicaciones, red e incluso firmware.

Backdoors (Puertas Traseras)

Mecanismo alternativo de acceso que omite controles de seguridad normales (contraseñas, MFA, firewall). Permite acceso prolongado incluso si la vulnerabilidad inicial ya fue corregida.

  • De cuentas: Usuarios ocultos con altos privilegios
  • De software: Apps modificadas con acceso secreto
  • En servicios: Daemons alterados para comandos ocultos
  • En red: Abren puertos o establecen shells reversas periódicas
  • Firmware/BIOS/UEFI: Sobreviven a formateos y reinstalaciones

Shell Reversa

Canal de acceso remoto donde la máquina víctima inicia la conexión hacia el atacante (no al revés). Tras conectarse, el atacante puede enviar comandos como si tuviera una terminal en la víctima.

  • Evita necesitar puertos entrantes (sortea NAT y firewalls)
  • Puede programarse para reconectar periódicamente
  • Se integra con tareas programadas o servicios para sobrevivir reinicios
  • Se camufla como tráfico saliente legítimo (HTTP, DNS)

Limpieza de Huellas

Técnicas para ocultar, alterar o eliminar evidencia de actividades maliciosas dentro de un sistema, con el fin de evadir detección y complicar el análisis forense.

  • Manipulación de logs: Borrar entradas de eventos o alterar timestamps
  • Eliminación de archivos temporales: Scripts, exploits, binarios usados
  • Ocultamiento de procesos: Uso de rootkits para esconder procesos maliciosos
  • Modificación de configuraciones: Deshabilitar auditorías futuras

Logs y Sistemas de Eventos

Logs: Archivos que registran eventos del SO, aplicaciones, dispositivos y servicios. Son la principal fuente de evidencia forense. Contienen: timestamps, identidad del usuario, tipo de evento, IPs de origen/destino y mensajes de error.


Sistemas de eventos: Más amplios que los logs; incluyen el servicio o plataforma que gestiona registros, permitiendo almacenarlos, consultarlos, filtrarlos y reenviarlos (ej. Windows Event Viewer, syslog en Linux).


Un atacante puede intentar: borrar entradas, desactivar el servicio de eventos, corromper la base de datos de eventos o manipular configuraciones para que ciertos sucesos no se registren.

Ataque Man-in-the-Middle (MitM) con Ettercap

Técnica donde el atacante se interpone en la comunicación entre dos partes, interceptando datos, modificando información en tránsito y redirigiendo la comunicación sin que las víctimas lo noten.


Ettercap es una herramienta open-source que permite: ataques MitM, sniffing de tráfico, ARP spoofing/poisoning, DNS spoofing y análisis de paquetes en tiempo real.

⚠ Riesgos y consecuencias

  • Robo de credenciales (usuarios, contraseñas)
  • Captura de cookies o tokens de sesión
  • Manipulación de contenido en páginas web
  • Redirección a sitios falsos (phishing avanzado)
  • Violación de confidencialidad en las comunicaciones
